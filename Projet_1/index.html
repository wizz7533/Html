Généralités HTML / CSS

Problèmes CSS courants

Le calcul par éfaut des tailles en CSS es content-box, il est conseillé de le passer en border-box

Les Tailles calculées: si vous voulez utiliser des cadratin (em), il est plus sûr d'utiliser les cadratins racine (rem) car ilssont toujours calculés par rapport au cadratin racine du user styleshett et non pas le cadratin calculé du conteneur.

Ajouter des polices de caractère

On utilisera les propriétés @font-face pour déclarer en CSS une polices de caractère dont les fichiers (eot, svg, ttf, woff, woff2) sont hébergés sur le site pour ne pas forcer l'utilsateur à télécharger et installer des fichiers. -> @fontface

Différentier les feuilles de style

Il est possible d'associer plusieurs feuilles de style à une page html, et de différencier les "media" (screen, print) qui les utiliseront.

Inclure le responsive

Généraliser le fait d'écrire son CSS responsive en mobile-first, car la majorité des utilisateurs utilisent un smartphone pour aller sur internet.

Il est plus simple de le faire de cette manière (mobile -> screen-first) que dans l'autre sens.

Images

on peut intégrer une image directement avec img, mais avec figure, on a à disposition une légende et de quoi styliser le containner de l'image et non l'image elle même
